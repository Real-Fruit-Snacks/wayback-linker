Wayback Linker

Turn fragile web references into durable Wayback Machine snapshots without leaving your note.

Download latest View source

Requires Obsidian 1.11.4 or newer

Built for research that lasts

Archive a note’s references in one deliberate pass.

Wayback Linker finds external HTTP links, asks Internet Archive for fresh captures, and rewrites successful targets in the active Markdown note. Existing Wayback URLs and image links are skipped automatically.

What it handles

Useful control without hiding the messy parts.

Fresh capture first

Requests a current Save Page Now snapshot and only marks it fresh when Wayback returns a recent timestamp.

Fallback when needed

Optionally uses the latest existing snapshot when a host blocks, throttles, or refuses a new capture.

Visible batch progress

See the active URL, fresh captures, fallbacks, failures, and retries in a live progress window.

One link or the whole note

Use the ribbon command for a batch, or right-click a single URL when you only need one snapshot.

Throttle aware

Waits and retries when Internet Archive reports an active-session limit, with configurable timing.

Native keychain storage

Internet Archive credentials stay in Obsidian SecretStorage; plugin data contains only selected secret IDs.

Workflow

From live URL to archived reference.

  1. 01

    Run the command

    Open a Markdown note and choose the ribbon action or command palette entry.

  2. 02

    Watch each capture

    Keep the progress window open, or close it and reopen it from the clickable status bar item.

  3. 03

    Keep durable links

    Successful original targets are replaced with their fresh or fallback Wayback snapshot URLs.

Credential security

Secrets belong in the keychain, not the vault.

Wayback Linker uses Obsidian’s native secret selector for Internet Archive credentials. On upgrade, legacy saved values are migrated into SecretStorage and removed from plugin data.

  • Secret values never enter data.json
  • Values are resolved at request time
  • Named secrets can be managed centrally in Obsidian

Installation

Add it to your vault.

Download the latest release files and place them together in the plugin directory shown here.

  1. Download main.js, manifest.json, and styles.css from the latest release.
  2. Create this directory inside your vault.
.obsidian/plugins/wayback-linker

Copy all three files there, reload Obsidian, and enable Wayback Linker under Community Plugins.

Open latest release

Clone the repository, install dependencies, verify the suite, and build the production bundle.

git clone https://github.com/Real-Fruit-Snacks/wayback-linker.git
cd wayback-linker
npm install
npm test
npm run build

Questions

Before you archive.

Does it replace every URL automatically?

It replaces external HTTP and HTTPS targets that receive a fresh capture or an enabled fallback. Failed captures remain unchanged. Existing Wayback links and image targets are skipped.

Why do some captures use a fallback?

Internet Archive may reject heavily captured hosts, limit active sessions, or be unable to save a page. When fallback is enabled, the plugin looks for the most recent existing snapshot instead.

Where do Internet Archive credentials live?

In Obsidian SecretStorage. The plugin saves only the selected secret IDs, never the credential values, in its settings file.

Can I archive just one URL?

Yes. Right-click an external URL in the editor and choose Archive link with Wayback Machine.
